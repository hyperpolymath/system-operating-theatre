== Overview

`big-up.sh` is designed to be a comprehensive system management and hardening tool providing:

* *Health Checks* — System health monitoring and diagnostics
* *Multi-stage System Updates* — Staged, rollback-capable system upgrades
* *Security Hardening* — Automated security configuration and compliance
* *Remote Management via PSSH* — Parallel SSH operations across fleet

== Technology Governance

This project adheres to the *Hyperpolymath Standard* for technology selection.

=== Preferred Stack

[horizontal]
Systems/CLI:: Rust
Application Code:: ReScript (compiles to JS)
Runtime:: Deno (replaces Node.js/npm)
Backend Services:: Gleam (BEAM or JS target)
Mobile:: Tauri 2.0+ / Dioxus (Rust-first, no Kotlin/Swift)
Configuration:: Nickel
State/Meta:: Guile Scheme

=== Security Requirements

* SHA-256+ for all cryptographic operations (no MD5/SHA1)
* HTTPS only — no plain HTTP
* No hardcoded secrets
* SHA-pinned dependencies
* SPDX license headers on all files

See link:.claude/CLAUDE.md[CLAUDE.md] for complete policy.
